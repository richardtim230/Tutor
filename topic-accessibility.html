Why Accessibility with CSS?

Accessibility means making your web content usable by everyone, including people with disabilities. CSS plays a vital role in improving accessibility by ensuring that content is readable, navigable, and interactive for all users.

  • Good color contrast helps users with low vision or color blindness.
  • Visible focus indicators support keyboard navigation.
  • Visually hidden elements help screen readers provide context.
  • Respecting reduced motion preferences helps users with vestibular disorders.

Color & Contrast

Ensure sufficient contrast between foreground and background colors for readability. WCAG recommends a contrast ratio of at least 4.5:1 for normal text.

High Contrast Example
HTML: 

<div class="contrast-demo">High Contrast Example</div>
CSS: 

.contrast-demo {
  background: #111827;
  color: #fbbf24;
  padding: 1.3rem;
  border-radius: 1rem;
  font-size: 1.1rem;
  font-weight: bold;
  text-align: center;
}

Tip: Use online tools like WebAIM Contrast Checker to verify ratios.

Focus Indicators

Always show a visible outline or highlight when form controls or interactive elements receive focus. This supports users navigating with a keyboard.

HTML: 

<input class="focus-demo" type="text" placeholder="Try tabbing here!">
CSS: 

.focus-demo {
  outline: none;
  border: 2px solid #d1d5db;
  border-radius: 0.8rem;
  padding: 0.8rem 1.2rem;
  font-size: 1rem;
  transition: border-color 0.3s, box-shadow 0.3s;
}
.focus-demo:focus {
  border-color: #6366f1;
  box-shadow: 0 0 0 3px #a5b4fc99;
}

Tip: Never remove outlines with outline: none; unless you replace them with a clear visual indicator.

Visually Hidden Content

Sometimes, you'll want to include extra labels or instructions for screen readers that are hidden visually but accessible to assistive technology.

HTML: 

<button>
  <span class="visually-hidden">Close dialog</span>
  <i class="bi bi-x-lg" aria-hidden="true"></i>
</button>
CSS: 

.visually-hidden {
  position: absolute !important;
  height: 1px; width: 1px;
  overflow: hidden;
  clip: rect(1px 1px 1px 1px); 
  white-space: nowrap;
}

Tip: Use .visually-hidden for extra context, instructions, or labels.

Respect Reduced Motion Preferences

Some users experience discomfort with excessive movement. Use the prefers-reduced-motion media query to minimize or disable CSS transitions and animations for them.

Hover Me!
HTML: 

<div class="reduced-motion-demo">Hover Me!</div>
CSS: 

.reduced-motion-demo {
  transition: transform 0.5s;
}
.reduced-motion-demo:hover {
  transform: scale(1.1) rotate(-2deg);
}
@media (prefers-reduced-motion: reduce) {
  .reduced-motion-demo {
    transition: none !important;
  }
}

Tip: Always respect user preferences for reduced motion!

Best Practices for CSS Accessibility

  • Always test your color contrast and font sizes for readability.
  • Provide clear focus indicators for all interactive elements.
  • Use .visually-hidden for extra labels and context.
  • Never use CSS to hide important content from assistive technology.
  • Respect prefers-reduced-motion for users who request it.
  • Test with keyboard and screen reader for real accessibility!

Practice: Make Accessible Buttons

Create a button with a visible focus indicator, high contrast, and a visually hidden label for screen readers.

HTML: 

<button class="accessible-btn">
  <span class="visually-hidden">Submit form</span>
  <i class="bi bi-send" aria-hidden="true"></i> Send
</button>
CSS: 

.accessible-btn {
  background: #111827;
  color: #fbbf24;
  border: none;
  border-radius: 0.7rem;
  padding: 1rem 2rem;
  font-size: 1.1rem;
  font-weight: bold;
  cursor: pointer;
  transition: box-shadow 0.3s, border-color 0.3s;
  outline: none;
}
.accessible-btn:focus {
  box-shadow: 0 0 0 3px #fbbf24aa;
  border: 2px solid #fbbf24;
}
.visually-hidden {
  position: absolute !important;
  height: 1px; width: 1px;
  overflow: hidden;
  clip: rect(1px 1px 1px 1px); 
  white-space: nowrap;
}

Try tabbing to your button, checking color contrast, and using a screen reader to test accessibility!
