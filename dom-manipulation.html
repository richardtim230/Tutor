What Is DOM Manipulation In JS?

Try to picture a webpage as a LEGO set: each tag in HTML is a block you can see and touch. The DOM—Document Object Model—is how JavaScript “sees” and “talks to” those blocks.

Here’s where variables and functions (from the previous topics) come together. You use variables to keep track of stuff—like what color a button should be or what score a user has. You use functions to make changes whenever a user clicks, types, or interacts. DOM manipulation is what lets you actually change what the user sees, in real time!

  • Variables store info about things (text, colors, scores...)
  • Functions say what should happen (change text, add scores...)
  • DOM manipulation puts it all together—making your site interactive!

Getting Your Hands On The Page: Selectors

Before you can change anything, you need to grab—or “select”—the right element. Here’s how you do it in JavaScript: 


// By ID
let mainTitle = document.getElementById('main-title');

// By class (gets only the first)
let firstBtn = document.querySelector('.btn');

// All matching (returns a list!)
let allBtns = document.querySelectorAll('.btn');

Example 1: Changing Heading Text

<h2 id="main-title">Welcome!</h2>
<button onclick="updateTitle()">Change Title</button>
<script>
function updateTitle() {
  let heading = document.getElementById('main-title');
  heading.textContent = "You clicked the button!";
}
</script>

Welcome!

Making Changes: Update, Add, Remove

Once you’ve got the right element, you can update almost anything: text, styling, add/remove elements, and more.

Change The Style


let box = document.getElementById("colorBox");
box.style.background = "yellow";
box.style.borderRadius = "1rem";

Add New Content


let list = document.getElementById("tasks");
let newItem = document.createElement("li");
newItem.textContent = "Finish JS section!";
list.appendChild(newItem);

Remove Something


document.getElementById("toRemove").remove();

Example 2: Change Color & Add Items


<div id="colorBox" style="width:100px;height:50px;background:#eee;"></div>
<button onclick="makeYellow()">Yellow Box</button>
<ul id="tasks"><li>Learn JS DOM</li></ul>
<button onclick="addTask()">Add Task</button>
<script>
function makeYellow() {
  document.getElementById('colorBox').style.background = 'gold';
}
function addTask() {
  let li = document.createElement('li');
  li.textContent = 'Practice selecting elements!';
  document.getElementById('tasks').appendChild(li);
}
</script>

Mini Practice: Try It Yourself!

Here are practical little snippets to train with—mix what you learned about variables and functions to update, add, and react to elements on a real page.

  • Create a button that changes the background color of the page (use a variable for the color).
  • Make a list where you can add your favorite foods using a form and button.
  • Write a function that hides an element (like a message) when a button is clicked.
  • Bonus: Combine all—color, list, hide, and user input. Can you build a “dashboard” panel?

Example: Hide & Show


<div id="messageBox">Hello—Hide Me!</div>
<button onclick="hideMsg()">Hide Message</button>
<button onclick="showMsg()">Show Message</button>
<script>
function hideMsg() {
  document.getElementById('messageBox').style.display = 'none';
}
function showMsg() {
  document.getElementById('messageBox').style.display = 'block';
}
</script>
Hello—Hide Me!

Tips: Best Habits For DOM (From Real Projects)

  • Always select elements carefully. If you plan to reuse them a lot, save them in a variable.
  • Wrap your page-changing logic in functions so it's easy to run over and over without repeating yourself.
  • Mix data (variables!) and actions (functions!) to make interactivity; don't hard-code everything.
  • Double-check spelling of IDs/classes—they must match between HTML and JavaScript!
  • Try to keep all your JS at the end of the file or in a separate .js file for cleaner code.
