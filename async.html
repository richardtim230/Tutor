Why Do We Need Asynchronous Code?

In JavaScript, most code runs one line after the next. But sometimes, you have to wait for something: a server response, a file to load, a timer to finish, or the user to click. You don’t want your whole program to pause and freeze while waiting.
Asynchronous ("async") code lets you keep your app smooth and responsive, even when waiting for slow things like the internet or a database.
Whenever you see fetch(), setTimeout(), or anything with "await", you’re dealing with async JavaScript. 

console.log("Start");
setTimeout(() => {
  console.log("This happens later");
}, 1000);
console.log("End");
// Output:
// Start
// End
// This happens later

Promises: What Are They?

A Promise is like a placeholder for a value you’ll have in the future—like a ticket for a bus that hasn’t arrived yet. You use promises to run code after something slow (like an HTTP request) finishes. 

let promise = new Promise(function(resolve, reject) {
  // Do something slow:
  setTimeout(() => {
    resolve("Done!");  // Could reject("Error!") if something fails
  }, 800);
});

promise.then(result => {
  console.log(result);  // Prints "Done!" after delay
});
A promise can be:
  • Pending: still waiting for work to finish
  • Resolved: finished successfully, delivered a value
  • Rejected: failed, returned an error
If you use fetch() for web requests or .then() anywhere, you’re already using promises.

Chaining .then() for Sequence

You can use .then() repeatedly to run code in order, passing results along, or to handle the next step after each async action. 

function waitAndAdd(n) {
  return new Promise(resolve => {
    setTimeout(() => resolve(n + 2), 1000);
  });
}
waitAndAdd(3)
  .then(res => {
    console.log(res);      // 5
    return waitAndAdd(res);
  })
  .then(next => {
    console.log(next);     // 7
  });
Why chain? When you want to do several async actions in a specific order. Each .then waits for the previous one to finish.

Async/Await: Smoother Syntax for Promises

async and await are new keywords that let you write async code that reads like regular, step-by-step code, but still doesn’t block your app while waiting. 

function wait(n) {
  return new Promise(resolve => {
    setTimeout(() => resolve(n * 2), 1000);
  });
}

async function example() {
  let x = await wait(5);
  console.log(x);  // prints 10 after delay
}
example();
  • Any function with async in front returns a promise.
  • Use await inside async functions to “pause” until a promise is done, then get its value.
This is the style you’ll use most for modern projects, especially with fetch() and API calls.

Error Handling with Promises & Async

Always handle errors—things go wrong! Use .catch() with promises, or try...catch blocks inside async functions. 

let failPromise = new Promise((resolve, reject) => {
  reject("Something went wrong!");
});

failPromise
  .then(res => { /* won't run */ })
  .catch(err => {
    console.log(err); // Handles error
  });
    

async function catchError() {
  try {
    let value = await Promise.reject("Oops!");
  } catch(e) {
    console.log("Caught: " + e);
  }
}
catchError();
With async/await, errors act like in normal code: use try/catch for clarity and control.
