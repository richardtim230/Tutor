What is HTML?

HTML stands for HyperText Markup Language. It is the backbone of every website you visit. HTML is not a programming language; rather, it is a markup language used to structure and describe the content of web pages. The browser reads HTML files and displays them as formatted text, images, links, forms, tables, and more.

HyperText means text that contains links to other texts or resources. Markup refers to the tags you use to “mark up” elements in your page (for example, making text bold or creating a heading).

When you open a website, your browser fetches HTML files and interprets them, painting the page you see. HTML works hand-in-hand with CSS (for styling) and JavaScript (for interactivity), but HTML always comes first.

Basic Structure of an HTML Document

Every HTML document follows a standard structure. This structure helps browsers understand your page, and it makes your code easier to read and maintain. Here’s the skeleton of a simple HTML file: 


<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8">
    <title>My First Webpage</title>
  </head>
  <body>
    <h1>Hello, World!</h1>
    <p>Welcome to my website.</p>
  </body>
</html>
  • <!DOCTYPE html> tells the browser to expect HTML5. Always include this at the very top.
  • <html> is the root element that wraps the entire content of your page.
  • <head> contains meta-information, such as the page title, character encoding, and links to stylesheets or scripts. The contents of <head> are not directly shown on the page.
  • <title> sets the text shown in the browser tab.
  • <body> contains all the visible content, such as headings, paragraphs, images, links, etc.

Try changing the content within the <body> tag to see how the page updates.

HTML Elements & Tags

HTML is written using elements, which are made up of tags. Tags are keywords wrapped in angle brackets. Most elements have an opening tag, some content, and a closing tag: 


<p>This is a paragraph.</p>
  • Opening tag: <p>
  • Content: The text (or other elements) inside the tag
  • Closing tag: </p>

Some elements are self-closing and don’t need a closing tag. For example, the image tag: 


<img src="logo.png" alt="Logo">

Here, src specifies the image file, and alt provides alternative text if the image can’t be displayed (or for screen readers).

Example: Nesting Elements


<p>This is a <strong>bold</strong> word inside a paragraph.</p>

You can place tags within other tags to combine formatting and structure.

Common HTML Tags & Their Uses

Tag Purpose Example
<h1>–<h6> Headings (largest to smallest) <h2>Chapter Title</h2>
<p> Paragraph of text <p>This is a paragraph.</p>
<a> Hyperlink to another page or site <a href="https://www.codecademy.com">Learn</a>

Example: Using Headings and Lists


<h1>Grocery List</h1>
<ul>
  <li>Rice</li>
  <li>Beans</li>
  <li>Yam</li>
</ul>

This produces a large heading and a bulleted list below it.

Attributes in HTML

Attributes provide extra information about HTML elements. They appear inside the opening tag, after the element’s name, and usually come in a name="value" pair. 


<a href="https://codecx.examguard.com.ng" target="_blank">Visit CODECX</a>
  • href (for links): Sets the destination URL.
  • target="_blank": Opens the link in a new browser tab.
  • alt (for images): Describes the image for screen readers and if the image can't load.
  • src (for images): The source file for the image.

Example: Adding Images


<img src="cat.jpg" alt="A cute cat">

Block vs Inline Elements

HTML elements behave either as block elements or inline elements.

  • Block elements always start on a new line and take up the full width available. Examples: <div>, <p>, <h1>.
  • Inline elements flow along with the text and only take up as much width as necessary. Examples: <span>, <a>, <strong>.

Example:


<p>This is a <strong>bold</strong> word inside a paragraph.</p>

The <p> is a block element, while <strong> is inline.

Tips for Writing Good HTML

  • Always close your tags properly. This makes your code easier to read and avoids browser quirks.
  • Use alt attributes for images to improve accessibility and SEO.
  • Organize your document with headings (<h1>–<h6>) for clear structure.
  • Indent your code so it's easy to follow, especially for nested elements.
  • Keep your content inside the <body> tag.
  • Use meaningful tag names. For example, use <nav> for navigation, <main> for the main content, and <footer> for the page footer. This is called semantic HTML, and it helps both browsers and humans understand your page.

Practice: Build Your First HTML Page

Try out this example. You can copy and paste it into a new file, save it as index.html, and open it in your browser:


<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8">
    <title>My Practice Page</title>
  </head>
  <body>
    <h1>Welcome to My Practice Page</h1>
    <p>This is my first HTML file.</p>
    <ul>
      <li>I am learning HTML</li>
      <li>HTML is fun!</li>
    </ul>
    <a href="https://codecx.examguard.com.ng" target="_blank">Learn More</a>
  </body>
</html>

Feel free to experiment! Change the heading, add more list items, or try adding an image with the <img> tag.
