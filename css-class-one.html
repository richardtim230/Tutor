Day 1 (Thursday): CSS Intro & Selectors

What's Happening Today?

Today, you'll dive into CSS (Cascading Style Sheets), which controls the visual presentation of your HTML pages. You'll learn how to apply styles, use different types of selectors, and understand the concept of the cascade and specificity.

Concepts – Pay Attention to This:

What is CSS?

  • CSS stands for Cascading Style Sheets.
  • It controls the visual appearance of HTML elements (colors, fonts, layouts, etc.).
  • It separates presentation (CSS) from content and structure (HTML), a concept called separation of concerns.

How to Include CSS:

  • Inline CSS: Styles applied directly in an HTML tag using the style attribute. Use sparingly (e.g., for quick tests).
  • Example: <p style="color: blue;">Blue text</p>
  • Internal CSS: Styles defined within a <style> tag in the <head> of your HTML file.
    • <head>
    <style>
        p {
            color: blue;
        }
    </style>
</head>
  • External CSS: Styles in a separate .css file, linked to HTML using <link>. This is the best practice for maintainability.
    • <head>
    <link rel="stylesheet" href="styles.css">
</head>
  • Note: External CSS is preferred for most projects as it keeps your HTML clean and makes styles reusable across multiple pages.

CSS Syntax:

  • A CSS rule consists of a selector and a declaration block.
  • Selector: Targets the HTML element(s) to style (e.g., p, .class, #id).
  • Declaration Block: Contains one or more declarations (property: value pairs) inside curly braces {}.
  • Example:
    • h1 {
    color: navy;
    font-size: 24px;
}

Basic Selectors:

  • Element Selector: Targets all instances of a specific HTML element (e.g., p, h1).
  • Example: p { color: green; } (styles all paragraphs).
  • Class Selector: Targets elements with a specific class attribute. Use a dot (.) before the class name.
  • Example: .intro { font-weight: bold; } (styles elements with class="intro").
  • ID Selector: Targets a single element with a specific ID. Use a hash (#) before the ID name.
  • Example: #main-title { text-align: center; } (styles element with id="main-title").
  • Note: IDs must be unique per page. Classes can be reused on multiple elements.

Cascade and Specificity (Basic):

  • Cascade: CSS rules are applied in order of appearance, with later rules overriding earlier ones if they target the same element.
  • Specificity: Determines which rule wins when multiple rules target the same element. ID selectors are more specific than class selectors, which are more specific than element selectors.
  • Example: If p { color: blue; } and .intro { color: red; } both apply to a <p class="intro">, the color will be red because class selectors have higher specificity.

Your Hands-On Activity: Apply Styles to Your Bio Page

Set Up Your CSS File:

  • Open your "my-first-website" folder in VS Code.
  • Create a new file called styles.css in the same folder as index.html.
  • Link your CSS file to index.html by adding this in the <head>:
    • <link rel="stylesheet" href="styles.css">

Try Inline and Internal CSS (Just to Understand):

  • Add inline CSS to one paragraph in index.html (e.g., <p style="color: blue;">).
  • Add internal CSS in the <head> of index.html using a <style> tag to style all <h1> elements (e.g., h1 { font-size: 32px; }).
  • Then, move those styles to styles.css to practice external CSS.

Style Your Bio Page:

  • In styles.css, add styles for:
    • h1: Change the text color (e.g., color: #2c3e50;) and center it (text-align: center;).
    • p: Set a font size (e.g., font-size: 16px;) and add some margin (margin: 10px 0;).
    • Add a class to one paragraph (e.g., <p class="intro">) and style it with a different color or font weight (e.g., .intro { color: #2980b9; font-weight: bold; }).
    • Add an ID to your main heading (e.g., <h1 id="main-title">) and style it uniquely (e.g., #main-title { text-decoration: underline; }).
  • Open index.html with Live Server to see your changes.

Day 2 (Friday): Colors, Text & Box Model

What's Happening Today?

You'll explore CSS colors, text styling properties, and the CSS Box Model, which is critical for understanding how elements are sized and spaced on a page.

Concepts – Pay Attention to This:

CSS Colors:

  • Named Colors: Simple names like red, blue, black.
  • Hex Colors: Six-digit codes (e.g., #FF0000 for red). First two digits are red, next two are green, last two are blue.
  • RGB/RGBA: RGB uses values 0-255 for red, green, blue (e.g., rgb(255, 0, 0) for red). RGBA adds alpha for opacity (0 to 1, e.g., rgba(255, 0, 0, 0.5) for semi-transparent red).
  • HSL/HSLA: Hue (0-360), Saturation (0-100%), Lightness (0-100%). HSLA adds alpha. Example: hsl(0, 100%, 50%) for red.
  • Example:
    • p {
    color: #3498db; /* Hex */
    background-color: rgba(255, 165, 0, 0.7); /* RGBA */
}

Text Styling:

  • font-family: Sets the typeface (e.g., font-family: 'Arial', sans-serif;). Always include a fallback font (e.g., sans-serif).
  • font-size: Sets text size (e.g., 16px, 1.2em, 1rem). Pixels are absolute, em/rem are relative.
  • font-weight: Sets boldness (e.g., normal, bold, 700).
  • text-align: Aligns text (left, center, right, justify).
  • line-height: Sets spacing between lines (e.g., 1.6 for 1.6 times the font size).
  • text-decoration: Adds underlines, overlines, or strikethrough (e.g., underline, none).
  • letter-spacing: Adjusts space between characters (e.g., 2px).
  • Example:
    • h2 {
    font-family: 'Georgia', serif;
    font-size: 24px;
    font-weight: bold;
    text-align: center;
    line-height: 1.4;
}

CSS Box Model:

  • Every HTML element is treated as a rectangular box with:
    • Content: The actual content (text, images).
    • Padding: Space between content and border.
    • Border: Surrounds padding (and content).
    • Margin: Space outside the border, separating the element from others.
  • box-sizing: Controls how width/height are calculated.
    • content-box (default): Width/height only include content.
    • border-box: Width/height include content, padding, and border (recommended).
  • Example:
    • .box {
    width: 200px;
    padding: 10px;
    border: 2px solid black;
    margin: 15px;
    box-sizing: border-box;
}
  • Note: Always use box-sizing: border-box; to make sizing predictable. Apply it globally with * { box-sizing: border-box; }.

Your Hands-On Activity: Style Your Recipe Page

Apply Text Styling:

  • Open your recipe.html and recipe-styles.css.
  • In recipe-styles.css, add styles for:
    • h1: Use a different font family (e.g., font-family: 'Georgia', serif;) and center it.
    • h2: Use a vibrant color (e.g., color: #e74c3c;) and add text-decoration: underline;.
    • p: Set font-size: 16px; and line-height: 1.6;.
    • ul li: Add letter-spacing: 1px; to ingredients for readability.

Apply Box Model Properties:

  • Add a class to your ingredients list (e.g., <ul class="ingredients">).
  • Style it with:
    • .ingredients {
    padding: 15px;
    border: 1px solid #ccc;
    margin: 10px 0;
    background-color: #f9f9f9;
    box-sizing: border-box;
}
  • Add padding and margin to your instructions list (ol) for spacing.

Experiment with Colors:

  • Try different color formats (Hex, RGB, HSL) for text or backgrounds.
  • Example: background-color: rgba(200, 200, 200, 0.2); for a subtle background.

  • Try changing values in DevTools (these changes are temporary) to see how they affect the layout.

Day 3 (Saturday): Backgrounds & Display

What's Happening Today?

You'll learn how to enhance your pages with background images and colors, style borders creatively, and control how elements are displayed on the page using the display property.

Concepts – Pay Attention to This:

Background Properties:

  • background-color: Sets the background color of an element.
  • background-image: Sets an image as the background (e.g., url('image.jpg')).
  • background-repeat: Controls if/how the image repeats (repeat, no-repeat, repeat-x, repeat-y).
  • background-position: Sets the starting position of the background image (e.g., center, top left, 50% 50%).
  • background-size: Controls the size of the background image (auto, cover, contain, or specific sizes like 100px 100px).
  • Shorthand background: Combines all background properties.
  • Example:
    • .hero {
    background: #f1f1f1 url('hero.jpg') no-repeat center/cover;
}

Border Properties:

  • border-style: Sets the style (e.g., solid, dashed, dotted).
  • border-width: Sets the thickness (e.g., 2px).
  • border-color: Sets the color (e.g., #333).
  • border-radius: Rounds corners (e.g., 10px).
  • Shorthand border: Combines style, width, and color.
  • Example:
    • .card {
    border: 2px solid #2980b9;
    border-radius: 8px;
}

Display Properties:

  • display: Controls how an element is rendered.
    • block: Takes up full width, starts on a new line (e.g., <div>, <p>).
    • inline: Takes only the width of its content, stays on the same line (e.g., <span>, <a>).
    • inline-block: Like inline, but allows setting width/height and respects padding/margins fully.
  • Example:
    • li {
    display: inline-block;
    margin-right: 10px;
}
  • Note: Use Chrome DevTools to inspect and toggle display properties to see their effects.

Your Hands-On Activity: Enhance Your Page

Add a Background Image:

  • Find a free, high-quality image online (e.g., from Unsplash) and save it in your project folder (e.g., images/background.jpg).
  • In recipe-styles.css, add a background image to a section (e.g., your ingredients or instructions).
    • .ingredients {
    background-image: url('images/background.jpg');
    background-repeat: no-repeat;
    background-position: center;
    background-size: cover;
    padding: 20px;
}

Style Borders:

  • Add a styled border to your table from Week 1 or a new section:
    • table {
    border: 1px solid #3498db;
    border-radius: 5px;
}
th, td {
    border: 1px solid #ddd;
    padding: 8px;
}

Experiment with Display:

  • Make your list items (<li>) in your ingredients or goals list appear horizontally:
    • ul li {
    display: inline-block;
    margin-right: 15px;
}
  • Compare inline vs. inline-block by toggling in DevTools.

Your Homework:

  • Plan Layout of Your Multi-Section Page: Revisit your multi-section page idea from Week 1 (e.g., portfolio, product page).
  • In your styles.css, start planning styles for each section:
    • Add background colors or images to <header> or <section>.
    • Style borders for visual separation.
    • Experiment with display to adjust how elements (e.g., navigation links) are arranged.
  • Save and view with Live Server to see the visual impact.
