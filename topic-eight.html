Embedding External Content in HTML

Sometimes, you may want to display content from other websites or sources inside your own webpage. This could be a YouTube video, a Google Map, a PDF file, or even another webpage. HTML provides several elements that allow you to embed this kind of content: <iframe>, <embed>, and <object>. Each has its own use cases and attributes.

  • <iframe>: This element displays another web page inside a rectangular area on your site. It is often used for videos, maps, or widgets. The src attribute sets the URL of the page to show.
  • <embed>: This tag is used to embed interactive content, such as PDFs or Flash files (though Flash is no longer widely supported). The src attribute sets the file to display.
  • <object>: This element can embed different types of media and documents, such as images, PDFs, or even web pages. It uses the data attribute to set the source file and the type attribute to specify the file type.

Common Attributes Explained

  • src: Sets the address (URL or file path) of the content to embed. Used with <iframe> and <embed>.
  • data: Used in <object>, specifies the file or page to display.
  • width and height: Set the size (in pixels) of the embedded content.
  • type: Used in <object> to specify the kind of file (like application/pdf for PDFs).
  • title: Adds a description for accessibility, especially for screen readers.
  • allowfullscreen: Lets content, such as videos, be viewed in full screen.
  • frameborder: Used with <iframe> to control the border around the frame (deprecated in HTML5).

More Examples: Using iframe, embed, and object

Embedding a YouTube video with <iframe>:


<iframe src="https://www.youtube.com/embed/xyz123abc"
  width="560"
  height="315"
  title="Bootcamp Introduction"
  allowfullscreen></iframe>

In this example, <iframe> loads a YouTube video. The title describes the content for accessibility, and allowfullscreen lets viewers expand the video.

Embedding a PDF with <embed>:


<embed src="syllabus.pdf" width="700" height="500" type="application/pdf">

The <embed> element displays a PDF file directly on the page. The type attribute tells the browser what kind of document is being shown.

Embedding content with <object>:


<object data="syllabus.pdf" type="application/pdf" width="700" height="500">
  <p>Your browser does not support PDFs. <a href="syllabus.pdf">Download the file.</a></p>
</object>

The <object> tag lets you embed files and provide fallback content if the browser cannot display the file. The fallback is shown inside the element.

Things to Consider: Embedding External Content

  • Only use external content from trusted sources, especially with <iframe>, to avoid security risks.
  • Always provide descriptive title attributes for accessibility.
  • Some browsers may block certain embeds for security reasons; always test your pages.
  • If the embedded content is important, provide fallback text or download links for users with unsupported browsers.
  • Use width and height to organize layout and avoid layout shifts.
  • Avoid using deprecated attributes like frameborder and prefer modern HTML5 attributes.

Clear explanations, good descriptions, and tested embeds help both new and experienced users work with external content confidently.

Best Practices for Embedding Content

  • Prefer <iframe> for embedding interactive web content like videos or maps.
  • Use <embed> or <object> for documents, and always test for browser support.
  • Always provide fallback content for users with limited browser capabilities.
  • Keep embedded content sized appropriately for both desktop and mobile views.
  • Avoid loading unnecessary external content that could slow down your page.
  • Update or check embedded links periodically to prevent broken or outdated content.

Practice: Embed Different Types of Content

Try adding these examples to your webpage and see how each embed works: 


<iframe src="https://www.youtube.com/embed/xyz123abc" width="560" height="315" title="Demo Video" allowfullscreen></iframe>

<embed src="document.pdf" width="600" height="400" type="application/pdf">

<object data="presentation.pdf" type="application/pdf" width="700" height="500">
  <p>Your browser does not support PDF viewing. <a href="presentation.pdf">Download here.</a></p>
</object>

Try different file types, sizes, and fallback options. Test your page in several browsers to see how embeds behave for different users.
