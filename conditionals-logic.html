Why Conditionals Matter Now

Up to this point, you’ve learned to work with variables, operators, loops, and events. By themselves, each lets you store, change, or repeat things in your code. What’s missing is the “brains”—making your code respond to situations. Conditionals add logic: they let your program decide and react, shifting between outcomes.
In real life, you make choices all the time:
  • If it’s raining, take an umbrella.
  • If the grade is >= 70, you pass.
  • Your phone battery is low? Plug it in.
Programming conditionals work on the same principle: check a condition, do something if it’s true (and something else if not).

Conditionals & Logic: What and Why?

Whenever you want your code to answer questions or handle options (“Should I show an error? Is this user logged in? Is this number even?”), you need conditionals. They unlock smarter functions, dynamic pages, and personalized interfaces. 

// Imagine a weather app
if (temperature < 20) {
  console.log("Wear a jacket");
}
That’s all there is: if (condition) { do something }

If Statements

The heart of all decisions in JavaScript. If something is true, the code block runs. 

let isLoggedIn = true;
if (isLoggedIn) {
  console.log("Welcome back!");
}
Why? If statements run only when a certain test is true. Your page can stay uncluttered and only display what matters for the situation.

Else and Else If

Use else if you want to provide a backup option. Add else if when you have several different checks you want to try in order. 

let score = 78;
if (score >= 90) {
  console.log("Grade: A");
} else if (score >= 70) {
  console.log("Grade: B");
} else if (score >= 50) {
  console.log("Grade: C");
} else {
  console.log("Grade: F");
}
Why? In real situations, you rarely have just two outcomes. With else if you can check several conditions in order.

Logical Operators in Action

Sometimes, you want to check more than one condition at a time. Use && for “and”, || for “or”, and ! for “not”. 

let user = "Sam";
let age = 20;

if (user && age >= 18) {
  console.log("Access granted");
} else {
  console.log("Access denied");
}
Why? With logic, your code can handle more realistic requirements and filter more precisely.

Switch Statements

For certain situations (like picking a menu or reacting to user input), switch statements give you a clean alternative. They check one value against different possibilities. 

let fruit = "banana";
switch(fruit) {
  case "apple":
    console.log("Red fruit");
    break;
  case "banana":
    console.log("Yellow fruit");
    break;
  case "grape":
    console.log("Purple fruit");
    break;
  default:
    console.log("Unknown color");
}
Why? Whenever you have a single variable with lots of possible cases, switch is simpler and easier to read than lots of else if statements.
