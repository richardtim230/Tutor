Progressive Enhancement & Graceful Degradation

Back to Resources Previous: Browser Compatibility & Polyfills

What Do These Terms Mean?

Not all users see the web the same way. Their browsers, devices, and accessibility needs are different. As a web creator, your goal is to make sure your site is usable for everyone, from the most basic browser to the most advanced.
Progressive enhancement means you build your site so the essential content and features work everywhere, and then add extra styles and features for browsers that can handle them.
Graceful degradation means you start with the full experience and make sure your site remains functional, even if parts of it aren’t supported.

Progressive Enhancement: Start Simple

Progressive enhancement starts with clean, accessible HTML. The most important content and actions must be available in any browser. Then, you can use CSS to make things look better for those who support it. 


<form>
  <label for="search">Search:</label>
  <input type="text" id="search" name="search">
  <button>Go</button>
</form>

Image Preview

This form can be used anywhere, even if a browser doesn't support advanced features. If you add CSS, it simply becomes easier to use and more pleasant to look at, but the basic function stays the same. 


form { max-width: 400px; margin: 1em auto; }
input, button { font-size: 1em; margin: 0.2em 0; }
button { background: #6366f1; color: #fff; border-radius: 0.3em; border: none; padding: 0.5em 1em; }

Progressive enhancement means your site works for everyone, and looks even better for those with newer browsers.

Graceful Degradation: Stay Functional

Graceful degradation is about planning for older browsers or situations where features may not work. You might use advanced HTML elements, but always ensure that there is a simple, usable fallback. 


<input type="date" name="birthday">

Image Preview

If a browser does not support <input type="date">, it will show a simple text field. Users can still type their birthday, even without the calendar picker. Always provide clear instructions or fallback text nearby if you rely on features that might not exist everywhere. 


<label for="birthday">Birthday (YYYY-MM-DD):</label>
<input type="date" id="birthday" name="birthday">

This way, your site remains usable and understandable no matter what browser is used.

Real-World Examples

  • Navigation: 
    
<nav>
  <ul>
    <li><a href="/">Home</a></li>
    <li><a href="/about">About</a></li>
    <li><a href="/contact">Contact</a></li>
  </ul>
</nav>

    A simple HTML navigation always works. Modern CSS can add horizontal layouts or hover effects, but the links remain usable everywhere.

  • Accessible Forms: 
    
<form>
  <label for="email">Email:</label>
  <input type="email" id="email" name="email">
  <button>Subscribe</button>
</form>

    Even if a browser doesn't check email validity, the user can still submit their address. The form always works.

  • Images & Alt Text: 
    
<img src="profile.jpg" alt="Profile picture of Richard">

    If the image doesn't load, users still know what it was supposed to show.

  • Tables for Data: 
    
<table>
  <caption>Class Schedule</caption>
  <tr><th>Day</th><th>Time</th></tr>
  <tr><td>Monday</td><td>10:00</td></tr>
  <tr><td>Tuesday</td><td>12:00</td></tr>
</table>

    Tables are readable in any browser. CSS can style them, but clear structure comes first.

Tips for Your Projects

  • Always write clear, semantic HTML before adding styles.
  • Use alt attributes for images and labels for form controls.
  • Test your pages in different browsers—especially older ones.
  • Check your pages with CSS turned off to see if the content is still usable.
  • Avoid relying on features that only exist in one browser.
  • Make instructions clear for form fields that use advanced types.
  • Use Can I Use to check if a feature is supported everywhere.

By focusing on content and accessibility first, your site will reach more people, no matter what device or browser they use.

Practice: Build and Test a Robust Page


<h2>Feedback</h2>
<form>
  <label for="message">Message:</label>
  <textarea id="message" name="message"></textarea>
  <button>Send</button>
</form>

Try this: Build the above page, view it with and without CSS, and test on different browsers. Is everything readable and usable? If you use advanced HTML elements, add instructions or alternative ways for users to interact.

Challenge for the class: Add a <input type="email">, style your form, and check how it looks on Chrome, Firefox, Safari, and a mobile browser. If you spot a difference, consider how you could guide users or add fallback instructions.
