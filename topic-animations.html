What Are Animations & Transitions in CSS?

Transitions and Animations bring web interfaces to life by adding movement and smooth changes to elements. Use transitions for gradual changes (like color, size, or position on hover), and animations for more complex, scripted sequences using keyframes.

  • Transitions create smooth effects when CSS properties change.
  • Animations let you define keyframes for multi-step, repeating or timed effects.
  • Both enhance user experience and draw attention to important elements.

CSS Transitions

transition lets you specify how CSS properties change over time. The effect starts when a property changes (often on :hover, :focus, or :active).

Hover Transition Example

Hover Me!
HTML: 

<div class="trans-demo">Hover Me!</div>
CSS: 

.trans-demo {
  background: #34d399;
  transition: background 0.6s, transform 0.6s;
}
.trans-demo:hover {
  background: #6366f1;
  transform: scale(1.12) rotate(-3deg);
}

Fade Effect Example

Fade Me!
HTML: 

<div class="fade-demo" id="fadeDemo">Fade Me!</div>
<button onclick="document.getElementById('fadeDemo').classList.toggle('hide')">
  Toggle Fade
</button>
CSS: 

.fade-demo {
  opacity: 1;
  transition: opacity 1s;
}
.fade-demo.hide {
  opacity: 0.2;
}

Tip: You can transition most numeric CSS properties, including color, background, transform, width, opacity, and more!

CSS Animations

Animations use @keyframes rules to create multi-step, repeating, and complex motion. You can animate properties like transform, opacity, color, and more.

Bounce Animation Example

I'm bouncing!
HTML: 

<div class="anim-demo">I'm bouncing!</div>
CSS: 

.anim-demo {
  animation: bounce 1.2s infinite alternate;
}
@keyframes bounce {
  from { transform: translateY(0);}
  to { transform: translateY(-24px);}
}

Custom Keyframes Example

Color Change!
HTML: 

<div class="animated-bg-demo">Color Change!</div>
CSS: 

.animated-bg-demo {
  animation: colorChange 2s infinite alternate;
}
@keyframes colorChange {
  0% { background: #fbbf24; }
  50% { background: #34d399; }
  100% { background: #6366f1; }
}

Tip: Control animation-duration, animation-iteration-count, animation-direction, animation-delay, and more!

Advanced Animation & Transition Techniques

Chaining Animations & Transitions

HTML: 

<div class="card">Hover me and shake!</div>
CSS: 

.card {
  transition: box-shadow 0.6s;
}
.card:hover {
  box-shadow: 0 4px 12px #6366f188;
  animation: shake 0.6s;
}
@keyframes shake {
  0% { transform: translateX(0);}
  25% { transform: translateX(-5px);}
  50% { transform: translateX(5px);}
  75% { transform: translateX(-5px);}
  100% { transform: translateX(0);}
}

Delays & Timing Functions

HTML: 

<div class="slow-fade">Slow Fade!</div>
CSS: 

.slow-fade {
  opacity: 1;
  transition: opacity 2s cubic-bezier(.36,2,.62,-1) 0.5s;
}
.slow-fade.hide {
  opacity: 0.2;
}

Tip: Use cubic-bezier for custom timing curves and animation-delay for sequenced effects!

Best Practices for Animations & Transitions

  • Use animations and transitions to guide user attention, not distract.
  • Keep motion short and smooth; avoid long delays or excessive repetition.
  • Test motion on all devices for smoothness and accessibility.
  • Avoid animating layout properties (width, height) if possible; use transform and opacity for performance.
  • Use prefers-reduced-motion media query for accessibility:

@media (prefers-reduced-motion: reduce) {
  .anim-demo, .bounce, .animated-bg-demo {
    animation: none !important;
    transition: none !important;
  }
}
