Why Variables & Data Types matter

Variables are the building blocks of programs — they store values that your code can read and change. Data types describe what kind of values variables hold (strings, numbers, booleans, objects, and more). Understanding differences (and pitfalls like coercion and hoisting) helps you write clearer, less-buggy JavaScript.

  • Declare with let, const, or var — prefer let and const today.
  • Know primitive types vs. objects (mutability, copying behavior).
  • Avoid surprises from automatic type coercion — use explicit conversion.

Variables: let, const, var

Use const for values that shouldn't be re-assigned, let for re-assignment, and avoid var (function-scoped and hoisted). const doesn't make an object immutable — it only prevents re-binding.

Data Types & Type Checking

JavaScript has primitives (string, number, boolean, undefined, null, symbol, bigint) and objects (arrays, functions, plain objects). Use typeof to inspect types, but note special cases like typeof null === "object".

Demo: typeof and implicit coercion
Watch how values convert to strings or numbers
Ready

// typeof examples
console.log(typeof 'hello'); // "string"
console.log(typeof 42);      // "number"
console.log(typeof null);    // "object" (quirk)

// coercion examples
console.log('5' + 3); // "53"  (string + number => string)
console.log('5' - 3); // 2     (string - number => number)

document.getElementById('typesDemoOutput').textContent = "'5'+3 => \"53\"  |  '5'-3 => 2";

Rule: When unsure, convert explicitly: Number('5'), String(5), Boolean(value).

Scope & Hoisting

Scope defines where variables are visible. let and const are block-scoped; var is function-scoped. Hoisting means declarations are moved to the top — but initializations are not. That’s why using variables before they’re initialized can surprise you.

Demo: Illustrating scope (open console)
Ready

// scope example
function demoScope() {
  if (true) {
    let x = 'block';
    var y = 'function';
  }
  console.log(typeof x); // "undefined" (x is block scoped)
  console.log(y);        // "function"  (y is function scoped)
}
demoScope();

document.getElementById('scopeDemoOutput').textContent = "See console for scope demo";

Careful: Avoid relying on var — prefer block-scoped variables to prevent accidental leaks.

Quick reference

Primitives
  • String, Number, Boolean
  • Null, Undefined
  • Symbol, BigInt
Objects
  • Array, Function, Date, RegExp
  • Plain object: {}
  • Mutable; beware shallow copies

Final tips & best practices

  • Prefer const when you don't plan to rebind; use let otherwise.
  • Convert values explicitly (Number, String, Boolean) to avoid coercion bugs.
  • Avoid var and global variables to reduce accidental clashes.
  • Test edge cases: empty arrays, NaN, null, and undefined.
