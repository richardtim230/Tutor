What Are Deprecated & Obsolete Elements?

Deprecated elements are tags or attributes in HTML that are no longer recommended for use. They are kept for backward compatibility, meaning older web pages might still work, but you should avoid them in new projects. Instead, use modern alternatives that follow current web standards.

Obsolete elements are tags and attributes that have been removed from the HTML standard. Browsers may ignore these tags or display them incorrectly, and they can make your code harder to maintain or understand.

Modern HTML encourages a clear separation between content (HTML), presentation (CSS), and behavior (JavaScript). This makes your pages easier to design, update, and make accessible to all users.

Common Deprecated HTML Elements & Attributes

Let's look at some examples of deprecated tags, what they do, and how to use the correct modern approach instead:

  • <font>: Used to set text color, size, and font face. Modern HTML uses CSS for all text styling. Example: 
    
<font color="blue" size="6" face="Arial">Hello World</font>
    Modern way: 
    
<span style="color: blue; font-size: 2em; font-family: Arial, sans-serif;">Hello World</span>
  • <center>: Centers content. Use CSS instead: 
    
<center>This text is centered</center>
    Modern way: 
    
<div style="text-align: center;">This text is centered</div>
  • <u>: Underlines text. CSS is preferred: 
    
<u>Important</u>
    Modern way: 
    
<span style="text-decoration: underline;">Important</span>
  • <big> and <small>: These change font size, but using CSS is better: 
    
<big>Large</big> <small>Small</small>
    Modern way: 
    
<span style="font-size: 1.5em;">Large</span> <span style="font-size: 0.7em;">Small</span>
  • Table attributes like bgcolor, border, and align: 
    
<table border="1" bgcolor="#f0f0f0" align="center">
  <tr><td>Cell</td></tr>
</table>
    Modern way: 
    
<table style="border: 1px solid #333; background: #f0f0f0; margin: 0 auto;">
  <tr><td>Cell</td></tr>
</table>

The modern method (using CSS) is more flexible, readable, and works better on all devices.

Obsolete HTML Elements (Never Use)

These tags have been completely removed from HTML standards. New browsers will ignore them, and their behavior is unpredictable.

  • <applet>: Used to run Java programs inside the browser. Java applets are no longer supported. Use JavaScript libraries such as Three.js or p5.js for interactive graphics and applications.
  • <isindex>: Provided a text search box. Modern HTML uses the <input> tag inside forms for all user input: 
    
<form>
  <input type="search" placeholder="Type to search">
</form>
  • <basefont>: Set the base font for a page. Use CSS to style fonts globally: 
    
<style>
  body { font-family: Verdana, sans-serif; }
</style>
  • <frame>, <frameset>, <noframes>: Used to split the browser window into sections. Modern layouts use CSS flexbox or grid for multi-column and multi-row layouts. Example using flexbox: 
    
<div style="display: flex;">
  <nav>Sidebar</nav>
  <main>Main Content</main>
</div>
  • <marquee>: Made text scroll across the screen. Modern effects use CSS animations: 
    
<div style="animation: scroll 5s linear infinite;">Scrolling text</div>
<style>
@keyframes scroll {
  0% { transform: translateX(100%);}
  100% { transform: translateX(-100%);}
}
</style>
  • <blink>: Made text blink. Not supported; use CSS if you need blinking effects: 
    
<span style="animation: blink 1s steps(2, start) infinite;">Blink!</span>
<style>
@keyframes blink {
  to { visibility: hidden; }
}
</style>

Always choose modern tags and CSS for layouts and effects. Libraries like Bootstrap, Tailwind CSS, and Bulma offer ready-made styles and layouts for beginners and advanced users.

Understanding Modern Alternatives & Tools

  • CSS (Cascading Style Sheets): The language for styling and layout. Instead of using old HTML tags for presentation, write CSS rules in a <style> block, an external .css file, or as inline style attributes.
  • Flexbox & Grid: These are modern layout techniques in CSS. display: flex; and display: grid; let you build responsive, multi-column or multi-row layouts without frames or tables.
  • CSS Libraries: Collections of ready-made styles. Bootstrap, Tailwind CSS, and Bulma make it easier to build attractive, responsive pages without writing much CSS yourself.
  • HTML Validators: Tools like W3C Validator help you check your code for errors, deprecated, or obsolete tags.

These tools and libraries are now the standard for professional web development. Beginners should get comfortable with CSS and try using a CSS framework to see how modern pages are built.

More Examples: Refactoring Legacy HTML

Original code using deprecated tags:


<font face="Verdana" color="red">Welcome to CODECX</font>
<center>
  <table border="2">
    <tr bgcolor="#eee">
      <td>Old cell</td>
    </tr>
  </table>
</center>

Refactored code using CSS and semantic HTML:


<span style="font-family: Verdana, sans-serif; color: red;">Welcome to CODECX</span>
<div style="text-align: center;">
  <table style="border: 2px solid #333;">
    <tr style="background: #eee;">
      <td>Old cell</td>
    </tr>
  </table>
</div>

Notice how the styling is moved to CSS, which makes it easier to update and maintain.

Using a library for button styles (Bootstrap example):


<link rel="stylesheet" href="https://cdn.jsdelivr.net/npm/bootstrap@5.3.0/dist/css/bootstrap.min.css">
<button class="btn btn-primary">Click Me</button>

Using Flexbox for layout instead of frames:


<div style="display: flex;">
  <aside style="width: 200px;">Sidebar</aside>
  <section style="flex: 1;">Main Content</section>
</div>

Things to Consider: Deprecated & Obsolete HTML

  • Replace old tags and attributes with modern HTML and CSS in all new code.
  • If you inherit legacy code, refactor it gradually using semantic tags and stylesheets.
  • Use validator tools to catch deprecated or obsolete markup.
  • Modernize your layouts with Flexbox or Grid, not tables or frames.
  • Try using CSS libraries to save time and maintain consistency.
  • Avoid mixing markup and styling. Keep your HTML clean and your styles in CSS.
  • Learn to recognize outdated tags so you can maintain and upgrade any web project confidently.

Best Practices for Modern HTML

  • Use semantic HTML elements (<header>, <nav>, <main>, <footer>) for structure.
  • Handle all visual presentation with CSS, not HTML attributes or tags.
  • Test your pages on different browsers, devices, and screen sizes.
  • Stay up to date with the latest HTML and CSS standards.
  • Use external stylesheets or libraries for scalable, maintainable designs.
  • Validate your code regularly and remove any deprecated or obsolete elements.

Practice: Refactor and Modernize HTML

Take a sample of old HTML and rewrite it using modern techniques: 


<font color="blue">Hello World!</font>
<center>Welcome to CODECX</center>
<marquee>Events coming up!</marquee>

Refactored version: 


<span style="color: blue;">Hello World!</span>
<div style="text-align: center;">Welcome to CODECX</div>
<div class="scroll-text">Events coming up!</div>
<style>
.scroll-text {
  animation: scroll 5s linear infinite;
}
@keyframes scroll {
  0% { transform: translateX(100%);}
  100% { transform: translateX(-100%);}
}
</style>

Try using a CSS library for buttons, cards, or layouts in your own projects. Practice spotting and updating deprecated code you might find online or in older tutorials.
