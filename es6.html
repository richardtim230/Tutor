var, let, and const - What’s the Difference?

In older JavaScript, every variable started with var. It works, but it can cause problems because it ignores modern “scope” rules.
Today, let and const replace var. They help you keep things tidy and prevent accidental mistakes (like overwriting the wrong variable or having one “stick around” too long).
Think of let as your “changeable” storage and const as your “locked box.”
Use let when you plan to update the value, const if you never want it to change. 

// Using var
var user = "Mimi";
user = "Sam";
var user = "Josh";   // This is allowed—confusing!

// Using let
let age = 20;        // lets you update (age = 21;)
let age = 25;        // Not allowed—no redeclaration in same scope

// Using const
const PI = 3.1416;   // never changes
// PI = 3.15;        // this would fail

let total = 10;
total = total + 5;   // allowed
Summary:
  • Use let if a value could change
  • Use const if a value should never change
  • Avoid var unless you’re fixing old code

What is Scope?

Scope means “where in your code does this thing exist?”
Use let and const inside functions, conditionals, or loops, and they’ll only be visible right where you defined them. That prevents bugs where two different pieces of code use the same variable by accident. 

function showMessage() {
  let msg = "Hello!";
  if (true) {
    let innerMsg = "Hi!";
    console.log(innerMsg); // shows "Hi!"
  }
  // console.log(innerMsg); // error: innerMsg is not visible here
  console.log(msg);        // shows "Hello!"
}
Tutor Note: Most bugs from var show up because it ignores block scope. Modern let and const always obey { }.

Block Scope Visualized

Whenever you write code inside { } braces (functions, if, for), variables made with let and const are NOT visible outside. 

for (let i = 0; i < 3; i++) {
  // i = 0, then 1, then 2
  console.log(i);
}
// console.log(i); // ERROR: i doesn't exist outside loop
If you used var instead, you could still access i outside the loop—which is rarely what you want.

const with Objects and Arrays

const prevents changing the variable itself, but you can still update what’s inside if it’s an object or array. 

const user = { name: "Zara" };
user.name = "Joy"; // allowed

const colors = ["red", "blue"];
colors.push("green"); // allowed
// user = {};         // ERROR: cannot reassign a new object
If you want the contents to never change either, you should use Object.freeze() (not common for beginners).

Arrow Functions

Arrow functions are a modern shortcut for creating functions, especially useful with arrays. They don’t have their own this, so don’t use them for object methods. 

// Function Declaration (classic)
function greet(name) {
  return "Hello, " + name;
}
greet("Tola");

// Arrow Function (modern)
const greet2 = (name) => {
  return "Hello, " + name;
};

// One-liner Arrow Function
const square = n => n * n;

let nums = [1,2,3,4];
let squares = nums.map(x => x * x); // [1,4,9,16]
If you want to access this inside an object method, use the old function syntax, not arrow functions.

All Together: Examples Compared

Need to double a number? Here are different ways: 

// Classic (declaration)
function double(n) { return n * 2; }
double(5);

// Expression (as a variable)
const double2 = function(n) { return n * 2; };
double2(7);

// Arrow function
const double3 = n => n * 2;
double3(9);

// Used in an array map
[2,4,6].map(n => n * 2); // [4,8,12]
