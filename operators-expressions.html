As you start combining and comparing data, JavaScript needs a way to “do things” such as math, text operations, or checking conditions. These are handled by operators. Whenever you see a plus, equals, or question mark in your code, you’re using them.

Operators & Expressions: What and Why?

Operators: Symbols that tell JavaScript “do something” (add, subtract, check equality, etc.)
Expression: Any chunk of code that produces a value. Think of it like a math problem or a question to JS. 

// Example expression
let price = 12 + 3;   // '+' is an operator, 12 + 3 is an expression, price gets 15

let isExpensive = price > 15; // '>' compares, returns true or false
Operators and expressions are everywhere, from variables to conditions, loops, and DOM updates.

Arithmetic Operators

Arithmetic operators let JavaScript perform basic math: + (add), - (subtract), * (multiply), / (divide), % (remainder). 

let a = 8, b = 5;
let sum = a + b;      // 13
let diff = a - b;     // 3
let prod = a * b;     // 40
let quot = a / b;     // 1.6
let rem = a % b;      // 3
Why? Any time you need to calculate values or change existing ones, arithmetic operators are your tools.

Assignment Operators

Assignment operators assign values to variables. Most common is =, but you’ll also see +=, -=, *=, /= (update a value with math at once). 

let counter = 10;
counter += 5;  // Same as: counter = counter + 5;  Now counter is 15
counter -= 3;  // counter = counter - 3;  Now counter is 12
Why? Assignment operators save you time and reduce confusion when updating variable values.

Comparison Operators

These help JS decide if something is the same or different.
  • == and ===: Equality (=== is “strict”—checks both value and type)
  • != and !==: Not equal
  • < > <= >=: Less/greater than

let x = 10;
let y = "10";
x == y      // true  (just value: same)
x === y     // false (type is different: number vs string)
x != 7      // true
x > 5       // true
Why? You use these operators every time you need to check conditions in if statements, loops, and validations.

Logical Operators

Logical operators let your code ask about more than one thing at once:
  • && (and): true only if both sides are true
  • || (or): true if at least one side is true
  • ! (not): reverses a true/false result

let age = 20;
let hasTicket = true;

if (age >= 18 && hasTicket) {
  // Both conditions true: you can enter
}
Why? Use these to combine multiple conditions in programs, like login forms or feature gating.

2. An age checker: Given a user’s number input, use an if/else statement and comparison operators to say “You are a minor” or “You are an adult.”
<!DOCTYPE html> <html lang="en"> <body> <input id="age" placeholder="Enter age"> <button onclick="checkAge()"/>Check</button> <div id="ageResult"></div> <script> function checkAge() { let age = Number(document.getElementById('age').value); if (age >= 18) { document.getElementById('ageResult').textContent = "You are an adult"; } else if (age > 0) { document.getElementById('ageResult').textContent = "You are a minor"; } else { document.getElementById('ageResult').textContent = "Please enter a valid age"; } } </script> </body> </html>
3. Two inputs: If both are filled, print “Both filled”; else, print “Please fill both.” Use logical AND.
<!DOCTYPE html> <html lang="en"> <body> <input id="a" placeholder="A"> <input id="b" placeholder="B"> <button onclick="checkBoth()"/>Check</button> <div id="bothResult"></div> <script> function checkBoth() { let a = document.getElementById('a').value.trim(); let b = document.getElementById('b').value.trim(); if (a && b) { document.getElementById('bothResult').textContent = "Both filled"; } else { document.getElementById('bothResult').textContent = "Please fill both"; } } </script> </body> </html>
