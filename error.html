Why Worry About Errors?

Errors are part of programming. Typos, invalid data, server issues, or unexpected input can and do happen. Ignore them and your page can break or your data may be lost. Good developers expect problems and write code that handles them in useful ways—showing messages, logging details, or using safe defaults.

Error Types in JavaScript

Common errors you’ll run into:
  • ReferenceError: You used a variable that wasn’t declared
  • TypeError: You tried something not allowed (like calling a non-function, or doing math with undefined)
  • SyntaxError: You wrote invalid JS code (missing a curly brace or comma)
  • RangeError: Number out of bounds, or .repeat() called with a huge number
  • Custom Error: Any error you define and throw yourself
Your browser’s Console panel will usually tell you what kind of error happened, and (roughly) where. 

// Examples:
let foo = 1;
console.log(bar);    // ReferenceError: bar is not defined
let x = null;
x();                // TypeError: x is not a function
let json = "abc";
JSON.parse(json);   // SyntaxError: Unexpected token a...
Don’t panic—read the error messages! They will usually show you the first place to look.

try...catch: Handling Mistakes in Code

try...catch lets you “try” running code, but if anything fails (throws), the browser will jump to your catch block to handle it. 

try {
  let json = JSON.parse('{"valid": true}');
  console.log(json.valid);
} catch (error) {
  console.log("Something went wrong:", error.message);
}
Your catch(error) block gets an error object, so you can print, log, or act based on what happened.
Place try...catch around only the code that might fail, not the whole app. Otherwise you might hide bugs.

Throwing Your Own Errors

You can make your program “fail on purpose” with throw—if a value isn’t what you expect, “throw” an error with your own message. 

function divide(a, b) {
  if (b === 0) throw new Error("Division by zero!");
  return a / b;
}
try {
  let result = divide(8, 0);
} catch(e) {
  console.log(e.message); // "Division by zero!"
}
Use this pattern to guard against bad input or impossible situations.

Debugging: Fixing Errors, Understanding Problems

Debugging is about rooting out mistakes and understanding where your code goes wrong. Tools and habits for catching and fixing issues:
  • Console.log: Print out values to follow what your code is doing
  • Browser DevTools: Check Console tab for errors, Sources for breakpoints (step-by-step), and Network for failed requests
  • Debuggers: The debugger; statement can pause JS anywhere for inspection
  • Isolation: Test small chunks of code in the console before adding to your main program
  • Read errors slowly: Don’t guess; error messages are clues
  • Comment out parts: Use // or /* ... */ to hide code and narrow the problem

// Example debugging with console.log
function greet(name) {
  console.log("Name received:", name);
  return "Hi, " + name;
}
greet("Gift");
DevTools: Right-click your page, pick "Inspect" > "Console" to open a panel and see errors live as you code.
