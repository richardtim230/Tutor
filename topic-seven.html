How to Use Images in HTML

Images help communicate ideas, illustrate concepts, and make web pages more engaging. In HTML, an image is added to a page with the <img> tag. This tag does not have a closing tag because it is a self-closing element.

  • <img>: This is the image element. It displays an image on the page. You must use the src attribute to specify the location of the image file.
  • src (source): This attribute provides the path or URL to the image file you want to display.
  • alt (alternative text): This attribute gives a description of the image. If the image fails to load, this description appears in its place. It is also read by screen readers to help users who cannot see the image.
  • width and height: These optional attributes set the size of the image in pixels. 

<img src="logo.png" alt="CODECX Institute Logo" width="200" height="70">

The example above displays the image file named logo.png with a width of 200 pixels and a height of 70 pixels. The alt text describes the image for anyone who cannot view it.

Adding Audio & Video in HTML

  • <audio>: This element lets you add sound or music to your page. Use the controls attribute to show play, pause, and volume buttons.
  • <video>: This element allows you to play videos directly on a web page. You can use width and height attributes to set the size, and controls to show playback buttons.
  • <source>: This tag is placed inside <audio> or <video> to specify the media file and its format. You can add multiple sources so the browser can choose the one it supports.
  • Fallback text: Any text placed between the opening and closing <audio> or <video> tags will be shown if the browser does not support those elements. 

<audio controls>
  <source src="bootcamp-intro.mp3" type="audio/mpeg">
  Your browser does not support the audio element.
</audio>

<video width="400" height="260" controls>
  <source src="interview.mp4" type="video/mp4">
  Your browser does not support the video tag.
</video>

These examples show how to add audio and video files that users can play, pause, or control the volume. Always provide fallback text for older browsers.

More Examples: Figures, Responsive Images, and Embeds

  • <figure>: This tag groups media (like an image) with a caption.
  • <figcaption>: The caption for a figure, placed inside <figure>.
  • srcset and sizes: These attributes in the <img> tag help display different image files based on the device screen size. This makes images load faster and look better on all devices.
  • <iframe>: This tag embeds another web page or content, such as a video, Google Map, or social media post, inside your webpage.
  • allowfullscreen: This attribute in <iframe> lets the embedded content be viewed in full screen.
  • title: This attribute in <iframe> describes the embedded content for screen readers. 

<figure>
  <img src="students.jpg" alt="Students presenting projects">
  <figcaption>CODECX students during their project showcase.</figcaption>
</figure>

<img
  src="banner-small.jpg"
  srcset="banner-large.jpg 1024w, banner-medium.jpg 640w"
  sizes="(max-width: 600px) 100vw, 1024px"
  alt="Bootcamp Banner">

<iframe
  src="https://www.youtube.com/embed/abc123xyz"
  width="560"
  height="315"
  title="CODECX Bootcamp Introduction"
  frameborder="0"
  allowfullscreen></iframe>

The examples above show how to group an image with a caption, serve different image sizes for different screens, and embed a video from YouTube.

Things to Consider: Images & Media Embeds

  • Always use meaningful alt text for images, so everyone understands your content.
  • Use width and height to help web browsers display images smoothly and avoid layout shifts.
  • Compress image and video files to make your site load faster.
  • Make media accessible by adding captions (<figcaption> or subtitles in video), and transcripts for audio.
  • Test your page on different devices to check how images and media look and work.
  • Use srcset and sizes for images that need to be responsive.
  • For third-party embeds like <iframe>, only use trusted sources and always describe the content with the title attribute.
  • Avoid autoplay for video and audio unless truly necessary; let users choose when to play media.

Good use of images and media helps both beginners and experienced users understand and enjoy your site.

Best Practices for Images & Media Embeds

  • Organize images in folders and use clear filenames so you can easily find and update them.
  • Use the loading="lazy" attribute in <img> tags to delay image loading until they are needed, which improves page speed.
  • Always check for broken image links and keep your media files up to date.
  • Use SVG for icons and graphics that need to scale without losing quality.
  • Make sure your media does not distract from your main content.
  • Provide alternatives for users who cannot see or hear your media.

Practice: Add Images and Media

Try using the tags and attributes described above to add images, audio, and video to your own HTML page: 


<img src="participant.jpg" alt="Bootcamp participant" width="250">

<audio controls>
  <source src="welcome.mp3" type="audio/mpeg">
</audio>

<video width="320" height="240" controls>
  <source src="demo.mp4" type="video/mp4">
</video>

<iframe src="https://www.youtube.com/embed/xyz123abc" width="560" height="315" title="Demo Video" allowfullscreen></iframe>

Try changing image sizes, adding captions, or using responsive image techniques. Practice describing your images clearly with the alt attribute.
