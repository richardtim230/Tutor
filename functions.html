Introduction: Why Functions?

Functions are the “recipe steps” in JavaScript applications. They help you organize your code into reusable blocks—so you don’t have to copy-paste logic everywhere.

Scope defines "where" in your code a variable or function is accessible. Understanding scope is crucial for writing bug-free apps and for using modern JavaScript well.

Declaring & Using Functions

There are several ways to create a function in JavaScript. All allow you to group code that you wish to re-use with different inputs:

1. Named Function Declaration


function greet(name) {
  return "Hello, " + name + "!";
}
console.log( greet("Alex") ); // Output: Hello, Alex!

2. Function Expression


const add = function(x, y) {
  return x + y;
};
console.log( add(5, 7) ); // Output: 12

3. Arrow Function (ES6+)


const double = (n) => n * 2;
console.log( double(8) ); // Output: 16

What Is Scope?

Scope controls where in your program variables and functions are accessible. Badly managed scope leads to confusing bugs, but good habits keep your code safe and readable.

Global Scope


let name = "Sam"; // Global variable

function display() {
  // Can use 'name' here
  alert("User: " + name);
}

Function (Local) Scope


function localExample() {
  let msg = "Inside function!";
  console.log(msg); // Accessible
}
console.log(msg); // ERROR: Not defined outside function

Block Scope (with let/const)


if (true) {
  let color = "purple";
  console.log(color); // Accessible here
}
console.log(color); // ERROR: Not defined outside if-block

Practical: Scope in Real Usage


let counter = 0; // Global

function increment() {
  counter++;
}
function demo() {
  let counter = 10; // Local to function
  counter++;
  console.log(counter); // Prints 11
}
increment();
console.log(counter); // Prints 1 (global 'counter')
demo();
console.log(counter); // Still prints 1

Try These: Function & Scope Challenges

  • Write a function multiply(x, y) that returns the result of multiplying two numbers. Call it with your favorite numbers.
  • Make a function called checkLogin(username) that shows "Welcome" or "Try again" based on a username you define.

  • Bonus: Try creating a function INSIDE another function, and see which variables are available (nested scope).

Bonus: Functions and Scope in HTML & CSS

<script>
function increaseFont() {
  let text = document.getElementById("demoText");
  let size = window.demoSize || 1;
  size += 0.15;
  text.style.fontSize = size + "em";
  window.demoSize = size;
}
</script>

Tweak the function to shrink text, or reset it! This mixes functions, variables, DOM and scope for practical hands-on experience.

Habits & Real-World Advice

  • Always use let or const for variables unless attached to window.
  • Use functions to break up big code into small chunks. Easier to debug!
  • Avoid naming conflicts by keeping variables scoped where needed.
  • If stuck, write out variable lifetimes on paper—it helps spot scope issues.
